# v0.2.0 开发计划：账单搜索和筛选

## 目标

v0.2.0 增加账单搜索和筛选能力，用于在账单列表页按条件查看 SQLite 中已有账单。

本版本不修改数据库表结构，继续基于现有 `transactions` 表实现：

- `id`
- `type`
- `amount`
- `category`
- `date`
- `note`
- `created_at`
- `updated_at`

筛选功能只影响账单列表展示，不影响现有新增、编辑、删除、首页统计、CSV 导出功能。

## 功能设计

账单列表页新增筛选区域，支持以下条件：

1. 按月份筛选。
2. 按收入 / 支出筛选。
3. 按分类筛选。
4. 按备注关键词搜索。
5. 按金额区间筛选。
6. 一键清空筛选条件。

默认状态为无筛选条件，显示全部账单，并保持当前按日期倒序的列表体验。

筛选条件可以组合使用。例如：

- 只看 2026-07 的支出。
- 只看分类为“餐饮”的支出。
- 搜索备注中包含“午餐”的账单。
- 查看金额在 10 到 100 之间的支出。

## TransactionFilter 设计

新增 C++ 筛选条件对象 `TransactionFilter`，建议放在：

- `src/models/TransactionFilter.h`
- 如有复杂转换逻辑，再增加 `src/models/TransactionFilter.cpp`

建议字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `hasMonth` | `bool` | 是否启用月份筛选 |
| `year` | `int` | 年份，例如 2026 |
| `month` | `int` | 月份，1 到 12 |
| `hasType` | `bool` | 是否启用类型筛选 |
| `type` | `TransactionType` | `income` 或 `expense` |
| `category` | `QString` | 分类，空字符串表示不筛选 |
| `noteKeyword` | `QString` | 备注关键词，空字符串表示不搜索 |
| `hasMinAmount` | `bool` | 是否启用最低金额 |
| `minAmount` | `double` | 最低金额 |
| `hasMaxAmount` | `bool` | 是否启用最高金额 |
| `maxAmount` | `double` | 最高金额 |

基础校验规则：

1. 月份启用时，`year` 必须大于 0，`month` 必须在 1 到 12。
2. 类型启用时，必须是 `income` 或 `expense`。
3. 金额启用时，金额不能小于 0。
4. 同时启用最低和最高金额时，`minAmount` 不能大于 `maxAmount`。
5. 分类和备注关键词在进入查询前统一 `trimmed()`。

QML 不直接拼 SQL，也不直接处理数据库查询细节。

## Repository 查询方案

在 `TransactionRepository` 中新增筛选查询方法：

```cpp
QList<Transaction> getTransactions(const TransactionFilter &filter);
```

查询原则：

1. 使用预处理 SQL，不拼接用户输入值。
2. WHERE 条件按启用的筛选项动态追加。
3. 月份筛选使用 `date >= :startDate AND date < :endDate`。
4. 类型筛选使用 `type = :type`。
5. 分类筛选使用 `category = :category`。
6. 备注关键词使用 `note LIKE :noteKeyword`，参数值为 `%keyword%`。
7. 金额区间使用 `amount >= :minAmount` 和 `amount <= :maxAmount`。
8. 排序保持 `ORDER BY date DESC, id DESC`。

月份边界：

- 2026 年 1 月：开始 `2026-01-01`，结束 `2026-02-01`。
- 2026 年 12 月：开始 `2026-12-01`，结束 `2027-01-01`。

备注关键词搜索需要处理 `%`、`_` 等 LIKE 通配符是否转义。v0.2.0 建议优先实现普通包含搜索；如果需要精确字面搜索，再增加 ESCAPE 规则。

现有方法保留：

- `addTransaction`
- `getAllTransactions`
- `getTransactionsByMonth`
- `getTransactionById`
- `updateTransaction`
- `deleteTransaction`

这样可以降低对首页统计、CSV 导出和已有列表逻辑的影响。

## TransactionListModel 修改方案

当前 `TransactionListModel.refresh()` 读取全部账单。v0.2.0 建议改为支持筛选状态：

新增成员：

- `TransactionFilter m_filter`
- `bool m_hasActiveFilter`

新增方法：

```cpp
Q_INVOKABLE QVariantMap applyFilter(const QVariantMap &filterMap);
Q_INVOKABLE void clearFilter();
Q_INVOKABLE void refresh();
```

行为设计：

1. `applyFilter` 接收 QML 表单数据，转换成 `TransactionFilter`。
2. 转换失败时返回中文错误信息，列表不刷新。
3. 转换成功后保存筛选条件，并调用 Repository 筛选查询刷新模型。
4. `clearFilter` 清空筛选状态，并刷新为全部账单。
5. `refresh` 根据当前是否有筛选条件决定调用筛选查询或全部查询。

这样编辑、删除账单后，如果用户仍在筛选状态，列表可以保持当前筛选结果；如果没有筛选，则继续显示全部账单。

## QML 筛选界面设计

筛选界面放在 `TransactionListPage.qml` 顶部，优先做简单表单，不做复杂弹窗和动画。

建议控件：

1. 月份输入框：`YYYY-MM`。
2. 类型切换：全部 / 支出 / 收入。
3. 分类输入或分类按钮组。
4. 备注关键词输入框。
5. 最小金额输入框。
6. 最大金额输入框。
7. “应用筛选”按钮。
8. “清空筛选”按钮。

交互规则：

1. 页面进入时默认显示全部账单。
2. 点击“应用筛选”后调用 `transactionListModel.applyFilter(...)`。
3. 筛选失败时在页面顶部或筛选表单下方显示中文错误。
4. 点击“清空筛选”后清空输入框，并调用 `transactionListModel.clearFilter()`。
5. 没有符合条件的账单时，复用空状态文案，建议显示“暂无符合条件的账单”。

移动端注意：

1. 筛选区域必须可滚动。
2. 输入框和按钮高度保持可点击。
3. 不引入复杂底部导航。
4. 不使用复杂动画。

## 数据流

```text
QML 筛选表单
  -> TransactionListModel.applyFilter(filterMap)
  -> TransactionFilter 校验和转换
  -> TransactionRepository.getTransactions(filter)
  -> SQLite transactions 表
  -> QList<Transaction>
  -> TransactionListModel
  -> QML ListView
```

清空筛选数据流：

```text
QML 清空筛选按钮
  -> TransactionListModel.clearFilter()
  -> TransactionRepository.getAllTransactions()
  -> TransactionListModel
  -> QML ListView
```

## 小任务拆分

建议每次只做一个小功能：

1. 新增 `TransactionFilter` 基础结构和校验，不接 QML。
2. 在 `TransactionRepository` 新增筛选查询方法，不改列表 UI。
3. 为 `TransactionListModel` 增加 `applyFilter` 和 `clearFilter`，先用简单参数验证。
4. 在 `TransactionListPage` 增加月份筛选 UI。
5. 增加收入 / 支出筛选 UI。
6. 增加分类筛选 UI。
7. 增加备注关键词搜索 UI。
8. 增加金额区间筛选 UI。
9. 增加清空筛选按钮和空状态文案。
10. 做桌面端手动回归测试。
11. 做 Android 构建验证。

## 风险点

1. 日期格式不一致会影响月份筛选。
   - 新增和编辑已要求严格 `YYYY-MM-DD`，但历史数据可能仍有旧格式。
2. LIKE 搜索中的 `%` 和 `_` 默认有通配符语义。
   - v0.2.0 可先接受普通 LIKE 行为，后续再做字面转义。
3. 动态 SQL 条件容易误拼。
   - 必须使用参数绑定，不允许拼接用户输入值。
4. 筛选状态和编辑 / 删除后的刷新需要一致。
   - `TransactionListModel.refresh()` 应保留当前筛选状态。
5. QML 表单输入为空时需要明确表示“不启用该筛选”。
6. 金额区间需要校验 `min <= max`，否则用户会看到空列表但不知道原因。
7. 首页统计和 CSV 导出不应复用列表当前筛选状态。
   - 首页继续查本月全部账单。
   - CSV 继续导出全部账单，除非后续单独设计“导出筛选结果”。

## 验收标准

1. 无筛选条件时，账单列表仍显示全部账单，按日期倒序。
2. 输入月份后，只显示该月份账单。
3. 选择收入后，只显示收入账单。
4. 选择支出后，只显示支出账单。
5. 输入分类后，只显示对应分类账单。
6. 输入备注关键词后，只显示备注包含关键词的账单。
7. 输入金额区间后，只显示金额在区间内的账单。
8. 多个筛选条件组合时，结果同时满足所有条件。
9. 点击清空筛选后，表单恢复默认，列表恢复全部账单。
10. 没有符合条件的记录时，显示明确空状态。
11. 新增、编辑、删除账单功能不受影响。
12. 首页本月统计不受列表筛选影响。
13. CSV 导出仍导出全部账单，不受列表筛选影响。
14. Android 构建通过。
