# MVP 功能清单

本文档定义个人记账 App 第一版 MVP 功能范围。开发时应同时遵守 `docs/development-rules.md` 中的技术栈、开发流程和边界约束。

## 1. MVP 目标

第一版目标是完成一个可本地使用的个人记账 App：

- 用户可以记录收入和支出。
- 用户可以查看本月收入、本月支出和本月结余。
- 用户可以查看、编辑、删除账单记录。
- 数据保存在本地 SQLite。
- 界面文案使用中文。
- 不做登录、不做云同步、不做支付功能。

说明：`docs/development-rules.md` 将“数据导出”列为后续扩展方向；本文件根据当前需求将“CSV 导出”纳入 MVP 范围。后续开发以本文档的 MVP 功能清单为准。

## 2. 功能总览

| 编号 | 功能 | 是否进入 MVP | 说明 |
| --- | --- | --- | --- |
| 1 | 首页统计 | 是 | 展示本月收入、本月支出、本月结余 |
| 2 | 添加账单 | 是 | 支持添加收入和支出 |
| 3 | 账单列表 | 是 | 展示历史账单记录 |
| 4 | 编辑账单 | 是 | 修改已有账单 |
| 5 | 删除账单 | 是 | 删除已有账单，删除前二次确认 |
| 6 | 本地 SQLite 保存 | 是 | 所有账单数据仅保存在本地 |
| 7 | 固定分类 | 是 | 第一版不做分类管理 |
| 8 | CSV 导出 | 是 | 将本地账单导出为 CSV 文件 |

## 3. 功能详情

### 3.1 首页统计

功能目标：

- 让用户打开 App 后快速了解本月收支情况。

页面展示：

- 本月收入总额。
- 本月支出总额。
- 本月结余。

计算规则：

- 本月收入：当前自然月内 `type = income` 的账单金额总和。
- 本月支出：当前自然月内 `type = expense` 的账单金额总和。
- 本月结余：本月收入减去本月支出。

验收标准：

- 首页能显示收入、支出、结余三个统计值。
- 新增、编辑、删除账单后，首页统计能反映最新数据。
- 没有账单时，三个统计值显示为 0。

### 3.2 添加账单

功能目标：

- 用户可以新增一条收入或支出记录。

表单字段：

- 类型：收入或支出。
- 金额：必填，必须是合法正数。
- 分类：必填，从固定分类中选择。
- 日期：必填，默认当天。
- 备注：选填。

交互规则：

- 保存成功后，账单写入本地 SQLite。
- 保存成功后返回首页或账单列表。
- 金额输入不允许为空、负数或非数字。

验收标准：

- 可以成功添加收入账单。
- 可以成功添加支出账单。
- 添加后的账单能在账单列表中看到。
- 添加本月账单后，首页统计同步变化。

### 3.3 账单列表

功能目标：

- 用户可以查看已经保存的账单记录。

展示内容：

- 类型。
- 金额。
- 分类。
- 日期。
- 备注。

列表规则：

- 默认按日期倒序展示。
- 日期相同时，较晚创建的记录排在前面。
- 列表为空时显示中文空状态文案。

验收标准：

- 可以看到所有已保存账单。
- 新增账单后列表出现新记录。
- 编辑账单后列表展示最新内容。
- 删除账单后列表不再展示该记录。

### 3.4 编辑账单

功能目标：

- 用户可以修改已有账单内容。

可编辑字段：

- 类型。
- 金额。
- 分类。
- 日期。
- 备注。

交互规则：

- 从账单列表进入编辑。
- 编辑页默认填充当前账单数据。
- 保存后更新本地 SQLite。
- 保存后返回账单列表或详情来源页。

验收标准：

- 可以修改收入账单。
- 可以修改支出账单。
- 修改金额、类型或日期后，首页统计同步变化。
- 修改后的内容在账单列表中正确展示。

### 3.5 删除账单

功能目标：

- 用户可以删除不需要的账单记录。

交互规则：

- 删除前必须弹出中文二次确认。
- 用户确认后才删除。
- 用户取消后保留原记录。
- 第一版使用物理删除。

验收标准：

- 可以删除指定账单。
- 删除后账单列表不再显示该记录。
- 删除本月账单后，首页统计同步变化。
- 取消删除时数据不发生变化。

### 3.6 本地 SQLite 保存

功能目标：

- 所有账单数据只保存在用户设备本地。

数据规则：

- 不依赖服务端。
- 不上传数据。
- 不做云同步。
- App 重启后，本地账单仍然存在。

建议数据表：`records`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | INTEGER | 主键，自增 |
| `type` | TEXT | `income` 或 `expense` |
| `amount` | INTEGER | 金额，单位为分 |
| `category` | TEXT | 分类名称 |
| `date` | TEXT | 日期，格式为 `YYYY-MM-DD` |
| `note` | TEXT | 备注，可为空 |
| `created_at` | TEXT | 创建时间 |
| `updated_at` | TEXT | 更新时间 |

验收标准：

- 新增账单能写入 SQLite。
- 编辑账单能更新 SQLite。
- 删除账单能从 SQLite 移除。
- 关闭并重新打开 App 后，历史账单仍可查看。

### 3.7 固定分类

功能目标：

- 第一版提供可直接使用的固定收入和支出分类。

收入分类建议：

- 工资。
- 奖金。
- 兼职。
- 投资。
- 其他收入。

支出分类建议：

- 餐饮。
- 交通。
- 购物。
- 居住。
- 娱乐。
- 医疗。
- 学习。
- 其他支出。

规则：

- 第一版不做新增分类。
- 第一版不做编辑分类。
- 第一版不做删除分类。
- 分类可以先在代码中固定维护。

验收标准：

- 添加收入时只能选择收入分类。
- 添加支出时只能选择支出分类。
- 切换账单类型时，分类选项同步变化。

### 3.8 CSV 导出

功能目标：

- 用户可以将本地账单导出为 CSV 文件，便于备份或自行分析。

导出范围：

- 第一版可导出全部账单。
- 如实现成本可控，可以支持按月份导出。

CSV 字段：

- 类型。
- 金额。
- 分类。
- 日期。
- 备注。
- 创建时间。
- 更新时间。

格式规则：

- CSV 表头使用中文。
- 金额导出为元，保留两位小数。
- 日期使用 `YYYY-MM-DD`。
- 类型导出为“收入”或“支出”。

验收标准：

- 用户可以触发 CSV 导出。
- 导出的 CSV 包含所有账单记录。
- CSV 表头为中文。
- 金额、类型、日期格式正确。
- 没有账单时，也能导出只有表头的 CSV，或给出明确中文提示。

## 4. 第一版不做的功能

第一版不做以下功能：

- 登录。
- 注册。
- 云同步。
- 支付功能。
- 复杂动画。
- 服务端接口。
- 远程数据库。
- 多设备同步。
- 复杂状态管理库。
- 自定义分类管理。
- 预算管理。
- 图表分析。
- 账单导入。
- 图片或发票上传。
- 多币种。
- 多账户管理。

## 5. 开发顺序建议

建议按以下顺序逐个小功能开发：

1. 初始化 Expo React Native + TypeScript 项目结构。
2. 接入 SQLite 并创建 `records` 表。
3. 定义账单类型和固定分类。
4. 实现添加账单。
5. 实现账单列表。
6. 实现首页统计。
7. 实现编辑账单。
8. 实现删除账单。
9. 实现 CSV 导出。

每次开发只完成一个小功能。修改代码前必须先说明准备修改哪些文件，修改完成后必须给出测试步骤。
