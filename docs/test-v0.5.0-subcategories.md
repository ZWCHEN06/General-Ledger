# v0.5.0 二级分类功能手动验收清单

## 验收范围

- 数据库 migration 与旧数据兼容
- C++ 模型、Repository、ListModel、AppController 接口
- QML 新增账单、编辑账单、分类管理、账单列表、筛选、CSV 导出
- Android 小屏幕页面可用性
- 周预算一级分类兼容性

## 验收前准备

- 当前代码在 `main` 分支。
- App 能正常编译并启动。
- 准备一个新数据库用于验证默认初始化。
- 准备一个已有旧数据库用于验证旧账单兼容。
- 建议至少保留以下一级分类：餐饮、交通、工资。

## 数据库

- [ ] `subcategories` 表存在。
  - 检查点：表包含 `id`、`category_id`、`name`、`is_default`、`sort_order`、`created_at`、`updated_at`。
  - 检查点：存在 `UNIQUE(category_id, name)`。
  - 检查点：`category_id` 外键指向 `categories(id)`。

- [ ] `transactions` 表存在二级分类字段。
  - 检查点：`subcategory_id` 允许为空。
  - 检查点：`subcategory` 允许为空。
  - 检查点：旧账单数据未丢失。

- [ ] 默认二级分类初始化正确。
  - 检查点：餐饮下存在早餐、午餐、晚餐、咖啡、零食、外卖、其他、请客吃饭。
  - 检查点：交通下存在地铁、公交、打车、火车、机票、停车、加油、其他。
  - 检查点：工资下存在基本工资、奖金、补贴、其他。

- [ ] 重复启动不会重复插入默认二级分类。
  - 操作：连续启动 App 两次。
  - 期望：同一个一级分类下默认二级分类名称不重复。

## C++ 层

- [ ] `Subcategory` 模型校验正确。
  - 检查点：`categoryId <= 0` 校验失败。
  - 检查点：空名称校验失败。
  - 检查点：错误信息为中文。

- [ ] `SubcategoryRepository` 查询可用。
  - 检查点：可以按 `categoryId` 查询二级分类。

- [ ] `SubcategoryListModel` 可供 QML 使用。
  - 检查点：`refresh(categoryId)` 后 rowCount 正确。

- [ ] `AppController` 二级分类接口可用。
  - 检查点：`refreshSubcategories(categoryId)` 成功刷新列表。
  - 检查点：`addSubcategory(categoryId, name)` 成功或返回中文错误。
  - 检查点：`updateSubcategory(id, name)` 成功或返回中文错误。
  - 检查点：`deleteSubcategory(id)` 成功或返回中文错误。

## 分类管理

- [ ] 可以从一级分类进入二级分类管理页。
  - 操作：进入分类管理，点击餐饮的“二级分类”。
  - 期望：打开二级分类管理页并显示“餐饮”。

- [ ] 可以新增自定义二级分类。
  - 操作：给餐饮新增“夜宵”。
  - 期望：新增成功并立即显示在列表中。

- [ ] 同一一级分类下不能新增重名二级分类。
  - 操作：再次给餐饮新增“夜宵”。
  - 期望：新增失败并显示中文错误。

- [ ] 不同一级分类下可以存在同名二级分类。
  - 操作：给交通新增“其他”或其他同名项。
  - 期望：只要当前一级分类下不重复即可新增。

- [ ] 可以编辑自定义二级分类。
  - 操作：把餐饮下“夜宵”改名为“宵夜”。
  - 期望：改名成功并刷新列表。

- [ ] 默认二级分类不能编辑。
  - 操作：尝试编辑餐饮下“早餐”。
  - 期望：不能编辑，或保存失败并提示“默认二级分类不能改名”。

- [ ] 可以删除未使用的自定义二级分类。
  - 操作：删除未被账单使用的“宵夜”。
  - 期望：删除成功并刷新列表。

- [ ] 默认二级分类不能删除。
  - 操作：尝试删除餐饮下“早餐”。
  - 期望：删除失败并显示中文错误。

- [ ] 已被账单使用的二级分类不能删除。
  - 操作：创建一笔餐饮 / 早餐账单后，尝试删除“早餐”或被使用的自定义二级分类。
  - 期望：删除失败并显示中文错误。

## 新增账单

- [ ] 新增支出账单可以选择二级分类。
  - 操作：新增支出，选择餐饮 / 早餐，金额 50。
  - 期望：保存成功，账单写入 `category_id`、`category`、`subcategory_id`、`subcategory`。

- [ ] 新增收入账单可以选择二级分类。
  - 操作：新增收入，选择工资 / 奖金。
  - 期望：保存成功，账单写入收入一级分类和二级分类。

- [ ] 切换一级分类时二级分类自动重置。
  - 操作：从餐饮切换到交通。
  - 期望：二级分类列表变为交通下的二级分类，并默认选中第一个。

- [ ] 一级分类下没有二级分类时不能保存。
  - 操作：构造一个没有二级分类的一级分类并新增账单。
  - 期望：保存失败并显示中文错误。
